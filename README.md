# N8N Deployment with Workers

Self-hosted N8N workflow automation with queue-based worker architecture.

## Architecture

- **PostgreSQL**: Data persistence
- **Redis**: Queue management
- **N8N Main**: Web UI and API server
- **N8N Workers**: Workflow execution (2 workers, scalable)

## Runtime Support

| Scenario | Host | Tool |
|----------|------|------|
| Docker | Linux / macOS | `docker compose` |
| Podman (Scenario 1) | Linux / macOS | `podman compose` or `podman-compose` |
| Podman Quadlets (Scenario 2) | **Linux + systemd only** | native systemd units |

---

## Quick Start

### Option A — Docker or Podman (podman-compose)

#### 1. Initial Setup

```bash
./scripts/setup.sh
```

The script auto-detects whether Docker or Podman is available and sets up accordingly.

This will:
- Detect the container runtime (Docker or Podman)
- Create necessary directories
- Set proper permissions (uses `podman unshare` for rootless Podman)
- Copy `.env.example` to `.env`
- Pull images

#### 2. Configure Environment

Edit `.env` and set your values:

```bash
nano .env
```

**Required changes:**

- `POSTGRES_PASSWORD`: Strong database password
- `N8N_PASSWORD`: Your N8N login password
- `N8N_HOST`: Your server IP address or domain
- `N8N_SECURE_COOKIE`: Set `false` for HTTP, `true` when behind an HTTPS reverse proxy

#### 3. Start N8N

```bash
# Docker
docker compose up -d

# Podman
podman compose up -d
# or
podman-compose up -d
```

#### 4. Access N8N

Open browser: `http://YOUR_SERVER_IP:5678`

Login with credentials from `.env` file.

### Option B — Podman Quadlets (Linux + systemd only)

> Quadlets are native systemd unit files generated by Podman. They require a Linux host with systemd (not macOS). Use Scenario A on macOS.

#### Prerequisites

- Podman >= 4.4
- Rootless Podman configured (`/etc/subuid` and `/etc/subgid` entries)
- systemd user session active

#### Setup

```bash
./init-project.sh --runtime=quadlets
```

Or run directly:

```bash
./podman/scripts/setup-quadlets.sh
```

This will:
1. Validate Podman version and rootless configuration
2. Create data directories
3. Install Quadlet unit files to `~/.config/containers/systemd/`
4. Enable linger for boot persistence
5. Start services in order: postgres → redis → n8n-main → workers

#### Managing Quadlet Services

```bash
# Check status
systemctl --user is-active n8n-postgres n8n-redis n8n-main 'n8n-worker@1' 'n8n-worker@2'

# Start / Stop / Restart
systemctl --user start n8n-main.service
systemctl --user stop n8n-main.service
systemctl --user restart n8n-main.service

# View logs
journalctl --user -u n8n-main --lines=30 --no-pager
journalctl --user -u 'n8n-worker@1' --follow

# Health check
./podman/scripts/health-check.sh

# Backup
./podman/scripts/backup.sh
```

## Management Commands (Scenario A)

### Start/Stop

```bash
# Start
docker compose up -d        # or: podman compose up -d

# Stop
docker compose down         # or: podman compose down

# Restart
docker compose restart      # or: podman compose restart
```

### Logs

```bash
# All services
docker compose logs -f

# Specific service
docker compose logs -f n8n-main
docker compose logs -f n8n-worker-1
```

### Status

```bash
# Container status
docker compose ps

# Health check
./scripts/health-check.sh

# Resource usage
docker stats      # or: podman stats
```

### Backup & Restore

```bash
# Create backup (Scenario A)
./scripts/backup.sh

# Create backup (Scenario B — Quadlets)
./podman/scripts/backup.sh

# Backups are stored in: ./backups/
```

## Scaling Workers

For Scenario A (compose), add more worker services in `docker-compose.yml`, or scale dynamically:

```bash
docker compose up -d --scale n8n-worker-1=4
```

For Scenario B (Quadlets), start additional worker instances:

```bash
systemctl --user start 'n8n-worker@3.service'
```

## Troubleshooting

### Check container logs

```bash
# Scenario A
docker compose logs n8n-main

# Scenario B
journalctl --user -u n8n-main --no-pager
```

### Restart specific service

```bash
# Scenario A
docker compose restart n8n-main

# Scenario B
systemctl --user restart n8n-main.service
```

### Reset everything (Scenario A)

```bash
docker compose down -v
rm -rf data/*
./scripts/setup.sh
```

## Security Notes

- Never commit `.env` file to Git
- Use strong passwords
- Consider using HTTPS with a reverse proxy (set `N8N_SECURE_COOKIE=true`)
- Regularly backup your data
- Keep container images updated

## Directory Structure

```
.
├── docker-compose.yml      # Services definition (Docker and podman-compose)
├── init-project.sh         # Entry point: auto-detect or --runtime=quadlets
├── .env                    # Environment variables (not in git)
├── .env.example            # Template for .env
├── data/                   # Persistent data (not in git)
│   ├── n8n/               # N8N workflows and credentials
│   ├── postgres/          # Database files
│   └── redis/             # Redis data
├── backups/               # Backup storage (not in git)
├── scripts/               # Scenario A management scripts
│   ├── setup.sh          # Initial setup (runtime-aware)
│   ├── backup.sh         # Backup automation
│   └── health-check.sh   # System health check
└── podman/                # Scenario B (Quadlets)
    ├── README.md          # Quadlet-specific documentation
    ├── quadlets/          # Systemd unit files
    └── scripts/           # Quadlet management scripts
        ├── setup-quadlets.sh
        ├── backup.sh
        └── health-check.sh
```
